## 三、视觉修复

### 3.1 Logo
- 替换为 `1000025693.png`（已去背景透明 PNG）
- 路径：`frontend/public/assets/image/head-logo.png`
- 首页居中显示，透明背景，无白框

### 3.2 SmartKB 浮球配色
- 背景：`linear-gradient(135deg, #7c3aed, #3b82f6)` + `backdrop-blur(20px)`
- 边框：`1px solid rgba(255,255,255,0.2)`
- 内发光高光
- 星轨：半透明白 + 品牌紫，双速旋转

### 3.3 头图加载优化
- 首张：`fetchpriority="high"` + `loading="eager"`
- 非首张：`loading="lazy"`
- `<head>` 预加载 API 响应 + 首张图片 URL
- 隐藏 `<img>` 提前触发浏览器下载

### 3.4 头图与内容过渡
- 头图底部 `mask-image: linear-gradient(to bottom, black 70%, transparent)`
- 内容区紧接头图，无间隙
- 数据栏卡片微上浮 8px，叠住头图底部

## 四、AI 协作开发故事板块

### 4.1 时间线概览

横向滚动条，展示 SuperRead 简报系统的开发历程：

```
需求分析(7/12) → 架构设计 → Qoder后端 → Trae前端 → Codex审查 → Bug修复迭代 → 上线(7/14)
```

每个节点可点击展开查看详情（Agent 名称 + 提交 hash + 耗时）。

### 4.2 Bug 案例卡片

数据来源：`docs/superpowers/bugs/` 目录（Markdown 文件）

每个卡片字段：
- **标题**：一句话描述问题
- **场景**：在做什么操作时出现
- **Agent**：谁发现的 / 谁修复的
- **根因**：技术层面的原因
- **修复方案**：怎么解决的
- **结果**：修复后效果
- **提交链接**：GitHub commit hash

示例：

```markdown
# SuperRead API Key 显示不全

- **场景**：用户在设置页查看 API Key，显示为 `sk-...3242`
- **Agent**：Claude Code 发现，Qoder 修复
- **根因**：`GetSettings` 用 `maskAPIKey()` 返回截断后的 key
- **修复**：移除 mask，前端加 ?full=true 获取完整 key
- **结果**：复制按钮拿到完整 key
- **提交**：`dd2d30b`
```

前端从 `docs/superpowers/bugs/` 读取 Markdown，渲染为卡片。新增案例只需添加 `.md` 文件。
