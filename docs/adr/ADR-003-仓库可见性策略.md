# ADR-003: 平台仓库公开只读、工具仓库开源

**日期**：2026-07-04  
**状态**：已采纳

---

## 背景

项目包含两类仓库：平台主项目和各个工具项目。需要决定各自的 GitHub 可见性。

## 决策

| 仓库 | 可见性 | 写权限 |
|------|--------|--------|
| `Oxelia51`（平台） | **公开（Public）** | 仅拥有者及授权协作者可推送 |
| 各工具项目 | **公开（Public）** | 各仓库独立管理 |

## 理由

- **技术展示**：平台代码本身也是技术实力的展示，公开有利于体现工程质量和架构设计
- **透明可信**：开源平台源码让用户了解数据流向，增强信任
- **写权限可控**：GitHub Public 仓库默认只有拥有者/协作者可推送，其他人只能 fork 和提交 PR，安全可控
- **学习价值**：平台作为学习实践项目，公开源码有利于交流和参考
- **展示价值**：多仓库公开共同构成完整的技术作品集

## 影响

- 平台仓库 `github.com/XiaoleC05/Oxelia51` 保持 Public，仅授权协作者可写
- 各工具仓库保持 Public，在 GitHub Releases 提供可下载的 exe 程序包
- 敏感信息（密钥、密码等）不得提交到仓库，通过 `.env` 和环境变量管理
- 如需限制某些实现细节，放入私有配置文件而非源码中
