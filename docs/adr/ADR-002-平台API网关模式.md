# ADR-002: 工具通过平台 API 网关调用（非页面跳转）

**日期**：2026-07-04  
**状态**：已采纳

---

## 背景

此前平台定位为"门户+目录"，用户点击工具后跳转到工具自身的页面。经讨论发现该方式存在问题：
- 用户体验割裂，每次使用工具都离开平台域名
- 各工具需重复实现认证逻辑
- 前端风格难以统一

## 决策

**平台统一承担所有工具的 UI 渲染，工具仅提供后端 API。**

| 层级 | 职责 |
|------|------|
| 平台前端（React） | 渲染所有页面，包括落地页、工具目录、每个工具的操作界面 |
| 平台后端（Go） | 用户认证、工具注册、API 网关：接收前端请求→转发到目标工具后端→返回结果 |
| 各工具 | 仅提供后端 API，无独立前端 |

## 理由

- **用户体验统一**：用户始终在 `oxelia51.com` 下操作，无跳转、无分离感
- **认证统一**：平台后端集中管理认证，工具 API 通过内网调用，不对外暴露
- **安全边界清晰**：工具 API 仅内网可达，公网无法直连
- **前端复用**：平台 React 的组件、样式、状态管理可跨工具共享
- **部署经济**：工具无需 Nginx/静态资源部署，只跑后端进程

## 影响

- 各工具项目不再需要构建前端，专注后端 API
- 平台前端需为每个工具开发对应的 UI 组件
- 平台后端需实现 API 网关转发逻辑
- 服务器 Nginx 配置：仅开放平台端口，工具端口仅内网监听
- 工具 GitHub Releases 可提供 exe 程序包（含简易本地界面），供下载本地使用
